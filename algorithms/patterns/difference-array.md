# Pattern：差分数组 / 前缀和（从 first principles）

> **如果整体 = 个体之和，且个体在「变化量」层面是稀疏的，就在变化量层面合并，最后再实例化。**

## 1. 把问题抽象掉

很多题表面是「数组操作」「区间修改」「最小化操作数」，剥掉包装后都是：

> 给定 n 个**子函数** `g₁(x), g₂(x), ..., gₙ(x)`（x 取离散值），
> 求 `min_x f(x)` 或 `f(x)` 在所有 x 的值，其中 `f(x) = Σ gᵢ(x)`。

任何「逐个体独立贡献 + 共享一个全局变量 x」的题，都能这样抽象。

## 2. 看子函数的结构

不同结构对应不同算法：

| 子函数 gᵢ 的结构 | 整体 f 的结构 | 适用算法 | 复杂度 |
|---|---|---|---|
| 任意 | 单调 | 二分 | O(log) |
| 任意 | 单峰 / 凸 | 三分 | O(log) |
| 分段常数，跳变点稀疏 | 也是分段常数 | **差分 + 前缀和** | O(n + 值域) |
| 分段线性 | 也是分段线性 | **二阶差分** | O(n + 值域) |
| 矩形覆盖 | 阶梯叠加 | **扫描线** | O(n log n) |
| 无结构 | 无结构 | 暴力 | O(n · 值域) |

⚠️ 二分 / 三分依赖**整体 f** 的单调或凸性；差分依赖**子函数 gᵢ** 的稀疏性。两条路互不相通。看到题先判断 f 的全局结构（单调？凸？），再看 gᵢ 的局部结构（稀疏？）。

## 3. 差分能用的两条核心性质

**性质 A：线性性**
$$\delta(g_1 + g_2 + \cdots + g_n) = \delta g_1 + \delta g_2 + \cdots + \delta g_n$$

「和的变化量」= 「变化量的和」。差分算子与求和**可交换**。

**性质 B：局部性**

每个 gᵢ 的变化量只在 O(1) 个点非零，且这些点只依赖 gᵢ 自身参数 —— 与其他子函数、与 x 轴别的位置无关。

合起来：**每个个体独立产出 O(1) 个事件，事件之间在同一根 x 轴上自由相加**。

## 4. 算法是结构的必然推论

```
Step 1（稀疏编码）：每个 gᵢ → O(1) 个 delta 事件，写进同一个 diff[]
                   利用局部性：每个体只 O(1)
                   利用线性性：直接累加，不冲突

Step 2（实例化）  ：对 diff[] 做一次前缀和，得到 f(x) 在所有 x 的值
                   唯一一次「逐 x 扫描」

Step 3（求目标） ：扫一遍找 min / max / specific x
```

总复杂度：O(n) 编码 + O(值域) 实例化 = **O(n + 值域)**。

**关键点**：编码阶段不知道任何 x 的具体值。我们在「稀疏事件空间」里做合并，只在最后一步才下降到「稠密值空间」。

## 5. 概念对偶

| 离散 | 连续 | 视角 |
|---|---|---|
| `diff[x]`（一阶差分） | `f'(x)`（导数） | 变化量 |
| 前缀和 | 积分 | 累计 |
| 差分法 | 微积分基本定理 | 编码 + 实例化 |
| 稀疏 delta 表示 | 测度（支撑稀疏） | 信息密度 |

「导数的和 = 和的导数」是连续版的同一定理。

## 6. 套路族（同源技巧）

都是「**用变化量代替全量**」的不同变体：

- **一阶差分**：区间加常数 → `diff[l] += c; diff[r+1] -= c`
- **二阶差分**：区间加等差序列 → `diff2` 处理首项 + 公差
- **扫描线**：区间覆盖问题 → 进出事件
- **懒标记线段树**：区间操作 defer 到查询时再实例化
- **FFT 卷积**：换基把「值域相乘」变成「频域相加」

## 7. 判题流程图

```
看到题
 ↓
能写成 f(x) = Σ gᵢ(x) 吗？
 ↓ 能
看 f(x) 整体结构
 ├─ 单调？  → 二分
 ├─ 单峰？  → 三分
 └─ 都不是  → 看 gᵢ 局部结构
              ├─ 分段常数 + 稀疏跳变 → 一阶差分
              ├─ 分段线性             → 二阶差分
              ├─ 区间覆盖类           → 扫描线
              └─ 真无结构             → 看能否换 x 域 / DP / 其他
```

## 8. 一句话总结

> **能用「变化量 / 事件 / 增量」描述的，就不要用「绝对值 / 状态 / 全量」描述。变化量是可交换的，全量不是。**
