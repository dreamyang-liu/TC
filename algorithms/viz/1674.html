LeetCode 1674：为什么 f(T) 不能二分？

① 每一对 (a, b) 的 cost：V 形

每对独立看：在 T = a+b 处 cost = 0；在 [a+1, b+limit] 区间内 cost = 1；其余 cost = 2。 所以单对的 cost 形状是「2 → 1 → 0 → 1 → 2」的 V 形（谷底有个小尖）。

② 总和 f(T)：多个 V 叠加 → 多峰多谷

把所有对的 cost 函数相加得到 f(T)。 每对的「谷」在不同位置时，叠加后会出现多个局部极小。 红点是所有局部极小，绿点是全局最优。

③ 为什么二分会失败

④ 差分动画：4 个 delta 如何「长」出 cost 函数

上图：diff 数组（大多为 0，只有 5 个非零端点 —— 基线 +2 加 4 个跳变 delta）。
下图：cost(T) 由 diff 的前缀和扫出来。竖线 = 当前扫描位置，红字 = 累加器当前值。
选「全部对」可以看多对的 delta 一起堆在 diff 数组里，前缀和就是 f(T)。

⑤ 正解：差分数组一次扫完

既然不能二分，那就老实枚举所有 T。直接枚举 + 对每对算 cost 是 O(n · limit)， 用差分数组把「区间贡献」摊到端点，就是 O(n + limit)： 

for each pair (a, b):       # a ≤ b
    diff[2]         += 2     # 默认 2 次
    diff[2*limit+1] -= 2
    diff[a+1]       -= 1     # [a+1, b+limit] 区间内只要 1 次
    diff[b+limit+1] += 1
    diff[a+b]       -= 1     # T = a+b 时为 0 次（在 1 的基础上再减 1）
    diff[a+b+1]    += 1

cur = 0; ans = n
for T in 2..2*limit:
    cur += diff[T]
    ans = min(ans, cur)
