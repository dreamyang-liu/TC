# 边界 Case Checklist（开做之前过一遍）

> **OpenAI / Anthropic 都把"边界处理"作为打分重点。** 别只验证 happy path。

每次开始写代码前花 30 秒过这张表，对应到当前题。

---

## 通用 7 大边界

| # | 边界 | 何时危险 |
|---|------|---------|
| 1 | **空输入** `[]`, `""`, `None` | 任何题 |
| 2 | **单元素** | 双指针、滑窗、分治 |
| 3 | **全相同 / 全不同** | 排序、去重、计数 |
| 4 | **极值** `INT_MAX`, `-INT_MAX`, 0 | 数值题、溢出 |
| 5 | **大小关系反转** `lo > hi`, `start > end` | 二分、区间 |
| 6 | **重复元素** | 二分边界、有序去重、哈希 |
| 7 | **越界访问** `arr[i+1]`, `arr[-1]` | 任何带索引的题 |

---

## 按数据结构分

### 数组 / 字符串
- [ ] `len == 0`
- [ ] `len == 1`
- [ ] 全 0 / 全负数 / 全相同
- [ ] 已排序 / 完全逆序
- [ ] 含**重复**元素（影响二分、去重）
- [ ] **Unicode / 多字节字符**（按 byte 还是 char？）
- [ ] **大小写敏感**？
- [ ] 是否包含空白 / 不可见字符？

### 链表
- [ ] `head is None`
- [ ] 只有一个节点
- [ ] 两个节点（许多反转 bug 在这）
- [ ] **环**（即使题目没说，问一下）
- [ ] head 本身可能被删除 → 用 **dummy**

### 树
- [ ] `root is None`
- [ ] 只有根
- [ ] 退化成**链**（左偏 / 右偏）
- [ ] **完全二叉树** vs 满二叉树 vs 平衡
- [ ] 节点值有**负数**（路径和题常坑）
- [ ] 节点值**重复**

### 图
- [ ] 自环（`u → u`）
- [ ] 重边（同对节点多条边）
- [ ] 不连通（多个连通分量）
- [ ] 单节点 / 空图
- [ ] **负权边** → Dijkstra 不能用
- [ ] **有环 vs DAG**

### 哈希表 / 集合
- [ ] key 是不可哈希类型？（list、dict）
- [ ] 哈希冲突 / 弱哈希攻击（在 production 题考虑）
- [ ] **迭代时修改**

### 矩阵
- [ ] `m == 0` 或 `n == 0`
- [ ] `1×N` / `N×1` 退化
- [ ] 非方阵
- [ ] 整行 / 整列相同

---

## 数值类专属

- [ ] **整数溢出**：Python 没有，但要说出来给面试官听
- [ ] **浮点精度**：`0.1 + 0.2 != 0.3`
- [ ] **除 0**
- [ ] **负数取模**：Python `-7 % 3 == 2`，Java/C++ 是 `-1`
- [ ] `mid = (lo + hi) // 2` —— Python 没问题，但说清楚 "在 C++ 要写 `lo + (hi-lo)//2` 防溢出"

---

## 字符串类专属

- [ ] **空串** vs **只有空格**
- [ ] **大小写**
- [ ] **首尾空白**（题目要不要 strip）
- [ ] **多语言字符**（汉字、emoji）`len()` vs 视觉宽度
- [ ] **转义字符**
- [ ] **delimiter 出现在数据里**（→ 长度前缀编码！经典 OpenAI 题）

---

## 设计 / 系统编码题专属

- [ ] **并发访问**：多线程读写同一个对象？要不要锁？
- [ ] **重复调用**：同一个 key 多次 set？
- [ ] **不存在的 key**：抛异常？返回 None？返回默认值？
- [ ] **容量耗尽**：满了之后？淘汰策略？拒绝？
- [ ] **持久化**：进程挂了数据丢吗？
- [ ] **错误恢复**：操作中途失败怎么办？
- [ ] **API 误用**：参数类型错？范围错？
- [ ] **时间 / 时区**：epoch、UTC、本地时间

---

## DP 专属

- [ ] `dp[0]` / `dp[0][0]` 的含义（空集？）
- [ ] **遍历顺序**和**依赖方向**对得上
- [ ] 滚动数组时**当前轮**不依赖被覆盖的值

---

## 何时该问，何时该假设

| 情况 | 做法 |
|------|------|
| 关键到影响算法选择 | **必问**（如：数据规模、是否有序） |
| 影响 1-2 行代码 | **声明假设**："我假设输入非空，如果可能为空我再加判断" |
| 显然的默认值 | 不问（如：数组下标从 0 开始） |

## 模板：面试中怎么演示边界处理

```python
def solve(arr):
    # Edge cases I'm handling explicitly:
    if not arr:                   # 空
        return ...
    if len(arr) == 1:             # 单元素
        return ...
    # ... 主逻辑
```

或者**在最后**统一加一段：

```python
# Sanity checks I'd add in production:
# - validate input types and ranges
# - guard against integer overflow (in other languages)
# - handle concurrent modification
```

**关键**：让面试官**看到你想了**这些，比真的写一堆判断更重要。
