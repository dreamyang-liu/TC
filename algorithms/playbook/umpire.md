# UMPIRE：60 分钟编码面试的解题流程

> **U**nderstand → **M**atch → **P**lan → **I**mplement → **R**eview → **E**valuate

OpenAI / Anthropic 都看重「**沟通与结构化思考**」远胜过「快速 AC」。沉默地刷题会**直接挂面**。

---

## U — Understand（5 分钟）

**目标**：把题目说回去，确认你和面试官在同一页上。

### 必问的 5 个澄清问题

1. **输入边界** — 大小？数据类型？范围？编码？
2. **输入有效性** — 会不会有空输入 / null / 重复 / 负数 / unicode？
3. **输出格式** — 返回什么类型？多个答案怎么办？顺序重要吗？
4. **场景** — 单次调用还是高频？数据规模？内存约束？
5. **隐含约束** — 是否要原地修改？能否用额外结构？并发？

### 不该跳过的小动作

- **大声复述**："所以这题是 …，对吗？"
- **手写两个例子**：一个 typical，一个 edge。
- **问一个"傻"问题**：往往面试官故意留了模糊点，问出来就是加分。

---

## M — Match（2-3 分钟）

**目标**：在脑里过一遍 pattern 库，匹配到最像的几个。

### 内心独白模板

> "听起来像是 X 类问题（最像 pattern A），但有 Y 这个特征所以可能要 pattern B。我先按 A 想……"

把候选 pattern **说出来**：面试官能从你的 candidate set 判断你的成熟度。

### 高频映射

| 关键词 | 优先考虑 |
|--------|----------|
| 子串 / 子数组 + "最长" / "和为" | 滑动窗口 / 前缀和 + 哈希 |
| 第 K 大 / 高频 K 个 | 堆 / 快选 |
| 下一个更大 / 接雨水 / 柱状图 | 单调栈 |
| 树 + 路径 / 最优值 | 后序 DFS，两个返回值 |
| "经过 X 步到达" / "最少操作" | BFS |
| 连通 / 朋友圈 / 合并 | 并查集 |
| 课程依赖 / 编译顺序 | 拓扑排序 |
| 重叠子问题 / 多种选择 | DP（先记忆化） |
| **设计 + 类** | **系统编码题** —— 先写接口 |

---

## P — Plan（5-8 分钟）

**目标**：在写代码**之前**让面试官点头。

### 三件事
1. **画图 / 写伪代码** —— 不要直接上 Python
2. **说复杂度** —— 时间 + 空间，给出推导而非答案
3. **请求确认** —— "这个方向 OK 吗？我开始实现？"

### 进阶：先暴力再优化

L5 标准的"展示思维过程"：

> "暴力解是 O(n²) ……（讲清楚）……，瓶颈在重复计算 X。我可以用哈希表把这步降到 O(1)，整体 O(n)。但内存换成 O(n)。这个 trade-off OK 吗？"

---

## I — Implement（20-30 分钟）

**目标**：写**可读**、**模块化**、**能跑**的代码。

### 写代码时的 5 个习惯

1. **变量命名清晰**：`left, right` > `l, r`；`window_count` > `cnt`
2. **小函数拆出来**：`def is_valid(...)`，主流程读起来像伪代码
3. **边写边讲**："我现在初始化 dp，dp[i][j] 代表……"
4. **不卡在小语法**：忘了 API 直接问，比 5 分钟 google 强
5. **写完一段就跑一个例子**（口头 trace 也行）

### Python 面试金句

```python
from collections import defaultdict, Counter, deque
from functools import cache       # 记忆化最方便
import heapq                       # 最小堆
from typing import Optional, List  # 显式类型注解，加分
from dataclasses import dataclass  # 类设计题神器
```

---

## R — Review（3-5 分钟）

**目标**：主动找 bug，比等面试官指出强一万倍。

### 必跑的 3 个测试

1. **基础用例**（题面给的）
2. **边界**（空、单元素、全相同、最大/最小）
3. **"诡异"用例**（负数、unicode、重复 key、并发……针对题目特性）

### 主动 self-critique

> "我注意到这里如果输入是空字符串，会进入死循环。我加一个 early return。"

---

## E — Evaluate（剩余 5-10 分钟）

**目标**：聊优化、变体、生产化。这是 L5 真正拉开差距的地方。

### 永远准备好的 5 个 follow-up

1. **更大数据**：从 10MB 涨到 10GB 怎么办？
2. **并发 / 分布式**：多线程？多机？数据一致性？
3. **持久化**：进程挂了能恢复吗？
4. **API 设计**：暴露给用户应该长什么样？错误码？
5. **监控 / 调试**：怎么知道它跑得对？怎么 debug？

**口诀**：每问一个 follow-up 都先答**"取决于 X"**，再展开两条路径。

---

## 一句话总结

> **沉默地写正确代码 ≠ 面试好。L5 的本质是「让面试官在脑海里也能复现你的思考过程」。**

每次练题，按这个流程走一遍。开始会慢，第 10 次就自然了。
