# 澄清问题清单（开始写代码前问什么）

> **澄清问题问得好 = 已经赢了 30%**。OpenAI 和 Anthropic 都把这当作 senior signal。

按题型 copy-paste 调整即可。

---

## 通用 4 问（任何题都问）

1. **数据规模？** "n 大概多大？10⁴ 还是 10⁹？"
2. **输入格式？** "保证有效吗？会不会有 null / 空字符串 / 负数？"
3. **输出要求？** "返回什么？多解时返回任意一个还是字典序最小？"
4. **性能约束？** "需要在线还是离线？延迟敏感吗？"

---

## 字符串题

- 字符集？ASCII / unicode / 多字节？
- 大小写敏感？
- 空格 / 标点要不要处理？
- 长度上限？
- 编码是 utf-8 吗？

## 数组 / 列表题

- 元素范围？有负数吗？
- 已排序吗？
- 可以修改原数组吗（in-place）？
- 重复元素怎么处理？
- 数组大小固定还是流式（stream）？

## 树 / 图题

- 节点数 / 边数？
- 有向还是无向？
- 有环吗？自环？重边？
- 边权范围？有负权吗？
- 连通吗？是 DAG 吗？
- 给的是邻接表还是邻接矩阵？

## 链表题

- 单向还是双向？
- 可能有环吗？
- 节点值范围？
- head 能不能为 None？

## DP / 优化题

- 求**最大值还是方案数**？
- 多个最优解返回哪个？
- 需要还原**路径**还是只要数值？
- 输入范围决定能不能 O(n²)？

## 系统设计 / 类设计题

- API 长什么样？方法签名给一下？
- 调用频率？读多还是写多？
- 错误怎么处理？抛异常还是返回错误码？
- 内存有限制吗？需要持久化吗？
- 是否需要线程安全？
- 是否需要分布式？

## 高频"陷阱"问题（问出来就赢）

### "delimiter 会出现在数据里吗？"
经典 OpenAI 题 KV serialize 的核心 —— 不能用简单分隔符，要用 length-prefix 编码。

### "时间戳保证单调递增吗？"
Time-based KV、profile trace 都会问。乱序输入解法完全不同。

### "key 大小写敏感吗？空白要 strip 吗？"
看似无害，错一个就全 WA。

### "保证有解吗？"
求最短路、求方案数，如果无解返回什么？

### "并发会修改这个数据结构吗？"
LRU、KV store 一定要确认 —— 决定要不要加锁。

### "是否要支持 undo / rollback？"
spreadsheet、SQL engine 这种系统题，问出来就是 senior 加分。

### "URL 里 fragment（#后面）要算同一个吗？"
Anthropic Web Crawler 题埋的雷。

### "递归调用算几层？"
栈追踪、profiler 题的核心 —— `{"a", "b", "a"}` 里两个 a 是不同 frame。

## 怎么"问得有水平"

❌ **新手问法**：
> "这题输入是什么？"

✅ **L5 问法**：
> "我假设输入是 List[int] 长度 ≤ 10⁴，元素在 ±10⁹ 范围，可能有重复，未排序。如果有不同我再调整。"

**先给假设，再问要不要调整**。比开放式提问效率高 5 倍。

---

## 问完之后必做

- 把澄清完的"题目契约"**重述一遍**："所以这题是 …，输入 …，输出 …，约束 …，对吗？"
- **手写一两个例子**，包括一个 edge case，让面试官点头：
  ```
  Input: []           → Output: ""
  Input: ["a"]        → Output: "a"
  Input: ["a", "b"]   → Output: "ab"
  ```
- **然后才开始 Plan 阶段**。
