树形 DP

后序遍历 + 在每个节点汇总子树信息，返回状态让父节点组合答案

核心思想

树形 DP 的套路：从叶子往根算（后序遍历）。每个节点处理完所有子节点后，把子节点的结果组合成自己的答案，再返回给父节点。

口诀：选当前 → 子不能选；不选当前 → 子取最大

四大模式

模式 1：单状态（最大深度）

最简单。每个节点返回"以我为根的子树深度"。

def maxDepth(root):
    if not root: return 0
    return 1 + max(maxDepth(root.left), maxDepth(root.right))

模式 2：两个返回值诀窍（直径 / 最大路径和）

关键区分：

- "经过我的路径"= 左链 + 右链 + 我 → 只用来更新全局 ans

- "给上面用的链"= max(左链, 右链) + 我 → 作为返回值给父节点

为什么？因为路径不能分叉！往上传的只能是一条直链。

def diameterOfBinaryTree(root):
    ans = 0
    def dfs(node):
        nonlocal ans
        if not node: return 0
        left = dfs(node.left)      # 左子树最长链
        right = dfs(node.right)    # 右子树最长链
        ans = max(ans, left + right)  # 经过我的路径（更新全局）
        return max(left, right) + 1   # 给上面用的（只能选一边）
    dfs(root)
    return ans

模式 3：选/不选模板（打家劫舍 III）

每个节点返回 (选我的最大值, 不选我的最大值) 元组。

def rob(root):
    def dfs(node):
        if not node: return (0, 0)
        left = dfs(node.left)
        right = dfs(node.right)
        rob_me = node.val + left[1] + right[1]      # 选我，子不能选
        skip_me = max(left) + max(right)             # 不选我，子随意
        return (rob_me, skip_me)
    return max(dfs(root))

模式 4：换根 DP（最难）

思路：如果要求"以每个节点为根时的答案"，暴力要 O(n^2)。

换根 DP 用两次 DFS：

1. 第一次 DFS：以节点 0 为根，算出 dp[0] 的答案

2. 第二次 DFS：从根往下，利用父节点的答案增量更新子节点的答案

关键公式：dp[child] = dp[parent] - child对parent的贡献 + parent对child的贡献

思考题

1. 为什么树形 DP 要用后序遍历？前序行不行？ ▶
后序遍历意味着"先处理子节点，再处理当前节点"。树形 DP 中，当前节点的答案需要子节点的结果才能计算，所以必须后序。前序是先处理自己再处理子节点，此时子节点信息还没算出来，无法组合。唯一例外是换根 DP 的第二次 DFS，那是用父节点信息更新子节点，所以用前序。
2. "经过当前节点的路径"和"延伸到当前节点的链"有什么区别？ ▶
"经过当前节点的路径" = 左子树最长链 + 当前节点 + 右子树最长链，形成一个倒 V 字形，可能是全局最优解。"延伸到当前节点的链" = max(左链, 右链) + 当前节点，是一条直线，只能选一个方向。区别本质：路径可以在当前节点"拐弯"（两边都用），但传给父节点时不能拐弯（只能选一边），否则父节点那里就会出现分叉。
3. 换根 DP 为什么只需要两次 DFS？ ▶
第一次 DFS 确定了一个基准答案（以某个节点为根）。换根时，从 parent 到 child 只发生了一个变化：child 的子树从 parent 的"下方"变成了"上方"。这个变化是 O(1) 可计算的增量。所以从根往下逐层传递，每个节点都能 O(1) 算出自己为根的答案。两次 DFS 总复杂度 O(n)，比暴力 O(n^2) 快得多。
