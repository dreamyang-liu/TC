二分查找 - 统一模板法

练习进度： 已完成 0/7

核心模板

所有二分查找题目，都可以套用同一个模板。记住这一个就够了：

left, right = 0, len(nums) - 1
while left < right:
    mid = (left + right) // 2
    if condition(mid):
        right = mid
    else:
        left = mid + 1
return left

核心思想：每次循环把搜索空间砍掉一半，最终 left == right 时就是答案。

为什么它是对的？

1. 不会死循环

mid = (left + right) // 2 是向下取整，所以 mid < right 恒成立。

这意味着 right = mid 一定能让区间变小，循环一定会终止。

2. 不会跳过答案

当 condition(mid) 成立时，right = mid（保留 mid），不是 mid - 1。

答案永远在 [left, right] 之内。

3. 结束时 left == right

循环条件是 left < right，退出时必然 left == right，不需要纠结返回谁。

统一套法表

你想找condition(mid)返回
最左的满足 XXleft
最右的满足 XNOT X（反转条件）left - 1

只要想清楚 "什么条件下 mid 可能是答案或者答案在左边"，就是 condition(mid)。

常见题目套用

题目condition(mid)说明
旋转数组最小值nums[mid] <= nums[right]mid 在右半段或就是最小值
第一个 >= targetnums[mid] >= target找最左满足
第一个 > targetnums[mid] > target找最左满足
最右的 <= targetnums[mid] > target反转 → 返回 left-1
求 sqrt(x)mid * mid >= x找第一个平方 >= x

取整方向规则

mid 的取整方向必须远离保留的那端。

标准模板用 right = mid 保留右端，所以 mid 向下取整（远离 right）。

如果你反过来写 left = mid 保留左端，就必须向上取整：mid = (left + right + 1) // 2。

记不住？就只用标准模板，不用记这条规则。

做题决策流程

我在找什么？ → 确定搜索空间（数组下标？答案范围？）
什么条件能排除一半？ → 写出 condition(mid)
找最左还是最右？ → 最左用标准模板返回 left；最右反转条件返回 left - 1
结束后检查合法性 → 答案可能不存在时，检查 nums[left] 是否真的满足要求

思考题

为什么 while left < right 不用 <=？如果用了会怎样？

如果用 <=，当 left == right 时循环还会继续执行。此时 mid == left == right，如果 condition(mid) 为真则 right = mid，区间没变小，死循环！用 < 保证退出时 left == right，既不死循环，又能定位答案。

如果把 right = mid 改成 right = mid - 1，什么情况会出错？

当 mid 本身就是答案时，right = mid - 1 会把答案排除出搜索空间。比如找第一个 >= 7，数组是 [1,3,5,7]，如果 mid 指向 7，设 right = mid - 1 就跳过了唯一正确答案。

"找最右的满足条件" 为什么可以转化为 "找最左的不满足条件 - 1"？

假设满足条件的元素是一段连续区间 [a, b]，那么第一个不满足条件的位置是 b+1。找到 b+1 后减 1 就是 b（最右满足的位置）。这利用了二分查找的有序性：满足条件的元素必然连续排列在一起。
