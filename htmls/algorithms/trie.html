Trie / 前缀树

每个节点代表"走到这里时的前缀"。共享前缀的字符串共享路径。

1. dict 版模板（最常用）

class Trie:
    def __init__(self):
        self.root = {}

    def insert(self, word):
        node = self.root
        for c in word:
            if c not in node: node[c] = {}
            node = node[c]
        node['#'] = True          # 结尾标记

    def search(self, word):
        node = self._walk(word)
        return node is not None and '#' in node

    def starts_with(self, prefix):
        return self._walk(prefix) is not None

    def _walk(self, s):
        node = self.root
        for c in s:
            if c not in node: return None
            node = node[c]
        return node
核心思想: 沿着字符一步步走，走到头能找到 # 标记就说明这个词存在。starts_with 只需要能走完就行，不要求有结尾标记。

2. 数组版（更快，固定字符集）

class TrieArray:
    def __init__(self):
        # 预分配节点池，children[i][c] = 下一个节点编号
        self.children = [[0]*26 for _ in range(100001)]
        self.is_end = [False] * 100001
        self.cnt = 0  # 当前已用节点数

    def insert(self, word):
        node = 0
        for ch in word:
            c = ord(ch) - ord('a')
            if self.children[node][c] == 0:
                self.cnt += 1
                self.children[node][c] = self.cnt
            node = self.children[node][c]
        self.is_end[node] = True
dict vs 数组:
- dict 版: 灵活，字符集不限，写起来快，Python 首选
- 数组版: 访问更快（O(1) 数组索引 vs 哈希），内存紧凑，C++/竞赛常用

3. 不写错要点

结尾标记不能省! 如果不加 #（或 is_end），插入 "apple" 后 search("app") 会返回 True —— 因为 "app" 是 "apple" 的前缀，路径确实存在，但 "app" 这个词并没有被插入过。

4. 经典套路

单词搜索 II（Trie + 回溯）

把所有待搜索词建成 Trie，然后在网格上 DFS。每走一步看 Trie 中有没有对应分支，没有就剪枝。比逐个词搜索快得多。

最大异或对（01 Trie）

把每个数的二进制表示插入 Trie（从高位到低位）。查询时贪心地走相反方向 —— 每一位尽量选和当前位不同的路径，这样异或结果该位为 1，最终得到最大值。
