单调栈 Monotonic Stack

栈里存"还在等下一个更大/更小元素"的下标。新元素来了就清理栈里被它"打败"的人。

核心模板

下一个更大元素 (Next Greater Element):

def next_greater(nums):
    n = len(nums)
    ans = [-1] * n
    stack = []  # 存下标，栈底到栈顶递减
    for i, x in enumerate(nums):
        while stack and nums[stack[-1]] < x:
            ans[stack.pop()] = x  # x 就是被弹出元素的答案
        stack.append(i)
    return ans
一句话理解：栈是"候选人等待区"。新人来了，比栈顶强的就把栈顶淘汰——"我就是你要找的更大元素！"

四种变体

只需改两个地方：比较方向 和 遍历方向。

目标栈的单调性比较条件遍历方向
下一个更大递减栈< x从左到右
下一个更小递增栈> x从左到右
上一个更大递减栈< x从右到左
上一个更小递增栈> x从右到左
口诀：找更大 → 栈递减；找更小 → 栈递增。
（栈和目标相反！想找大的，栈里存的是从大到小排列的候选人）

进阶：贡献法

经典应用：柱状图中最大的矩形（LC 84）。对每根柱子，用单调栈同时找到它左边和右边第一个比它矮的柱子，就能算出以它为高度的最大矩形宽度。

# 贡献法：每个柱子的贡献 = height[i] * (right[i] - left[i] - 1)
def largestRectangleArea(heights):
    heights = [0] + heights + [0]  # 哨兵
    stack = []
    ans = 0
    for i, h in enumerate(heights):
        while stack and heights[stack[-1]] > h:
            mid = stack.pop()
            ans = max(ans, heights[mid] * (i - stack[-1] - 1))
        stack.append(i)
    return ans

思考题

1. 为什么单调递减栈能找"下一个更大"？
因为栈维护的是一个递减序列（从底到顶），当新元素 x 比栈顶大时，说明 x 是栈顶第一次遇到的比它大的元素。栈顶一直在"等"，终于等到了打败它的人。而如果 x 比栈顶小，x 自己也还在等，入栈继续排队。
2. 栈里存下标而不是值有什么好处？
存下标可以通过 nums[stack[-1]] 随时拿到值，同时还能利用下标信息——比如计算距离（LC 739 每日温度）、判断窗口范围（单调队列中的过期检查）、计算矩形宽度（LC 84）等。存值就丢失了位置信息。
3. 柱状图最大矩形中，为什么要在末尾加一个0？
加末尾的 0 是为了确保遍历结束后栈中所有剩余元素都被弹出（因为 0 比任何正数都小）。加开头的 0 是为了在弹出时 stack[-1] 不越界，充当左边界哨兵。这样就不需要额外处理栈为空的边界情况。
