三大模式

1两数之和（一边遍历一边查） 
def twoSum(nums, target):
    seen = {}
    for i, x in enumerate(nums):
        if target - x in seen:      # 先查
            return [seen[target - x], i]
        seen[x] = i                    # 后存

关键：先查后存，防止自己配自己。

2前缀和 + 哈希表（和为K的子数组） 
def subarraySum(nums, k):
    prefix = {0: 1}  # 初始化！空前缀和为0出现1次
    curr_sum = count = 0
    for x in nums:
        curr_sum += x
        count += prefix.get(curr_sum - k, 0)
        prefix[curr_sum] = prefix.get(curr_sum, 0) + 1
    return count

核心思想：curr_sum - prev_sum = k，即 prev_sum = curr_sum - k。

进阶：滚动哈希（Rabin-Karp）

用多项式哈希在 O(1) 时间内从一个窗口滑动到下一个窗口。

# 字符串匹配：O(n+m) 平均
def rabinKarp(text, pattern):
    base, mod = 31, 10**9 + 7
    m = len(pattern)

    # 计算pattern的哈希
    pat_hash = 0
    for c in pattern:
        pat_hash = (pat_hash * base + ord(c)) % mod

    # 滑动窗口
    win_hash = 0
    power = pow(base, m - 1, mod)
    for i, c in enumerate(text):
        win_hash = (win_hash * base + ord(c)) % mod
        if i >= m:
            win_hash = (win_hash - ord(text[i - m]) * power * base) % mod
        if i >= m - 1 and win_hash == pat_hash:
            if text[i-m+1:i+1] == pattern:  # 验证（防哈希碰撞）
                return i - m + 1
    return -1

思考题

为什么两数之和要"先查后存"？举一个如果"先存后查"会出错的例子。

例子：nums = [3, 3], target = 6。

如果"先存后查"：遍历到第一个3时，先存 seen[3]=0。然后查 target-3=3，发现 seen 里有！返回 [0, 0]。
但这是同一个元素，不是两个不同的数。

"先查后存"：遍历到第一个3时，先查 seen 里有没有6-3=3，没有。再存 seen[3]=0。
遍历到第二个3时，先查 seen 里有没有6-3=3，有！返回 [0, 1]。正确！

前缀和+哈希表中，{0: 1} 初始化的含义是什么？不加会漏掉哪些情况？

{0: 1} 表示"前缀和为0出现过1次"，对应的是空前缀（一个元素都不取）。

如果不加：当 curr_sum 恰好等于 k 时（即从数组开头到当前位置的子数组和就是k），
我们需要找 curr_sum - k = 0 在 prefix 中出现的次数。如果没有初始化 {0:1}，就找不到，漏掉了这种情况。

例：nums = [1, 2, 3], k = 3。前缀和到第2个元素时是3，需要 prefix[0] = 1 才能数到子数组 [1,2]。

Counter 的减法 c1 - c2 为什么只保留正数？这在什么场景下有用？

设计哲学：Counter 代表"多重集合"，元素个数不能为负。减法表示"从集合中移除"，移除比拥有的多就是0。

有用场景：
- 判断一个单词能否由另一个的字母组成：Counter(word) - Counter(available) 为空则可以。
- 找缺少的字母：Counter(required) - Counter(have) 得到还差什么。
- 滑动窗口中比较字符频率。
