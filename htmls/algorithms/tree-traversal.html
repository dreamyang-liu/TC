二叉树递归 / 遍历

树 = 根 + 左子树 + 右子树。递归函数只想清楚：当前节点做什么，子树返回什么。

三种遍历

  • 前序 (根左右): 先处理自己，再递归子树。适合自顶向下传递信息。
  • 中序 (左根右): BST 中序遍历得到升序序列。
  • 后序 (左右根): 先递归子树，再处理自己。适合自底向上聚合信息。

何时用哪种？

  • 需要从上往下传参数 → 前序
  • BST 相关/有序性 → 中序
  • 需要子树结果来计算当前节点 → 后序

递归三问

  1. 返回什么？ 子树要向上汇报什么信息？
  2. 递归基返回什么？ 空节点 / 叶节点返回什么？
  3. 怎么用子树返回值组合？ 左右子树的返回值如何合成当前节点的答案？

两个返回值诀窍

很多树题需要维护两个量：

  • "经过我的" — 在递归过程中内部更新全局答案（如最长路径经过当前节点）
  • "给上面用的" — 作为返回值传给父节点（如以当前节点为端点的最长路径）

典型例题：124. 二叉树中的最大路径和、543. 二叉树的直径

进阶遍历方式

  • 迭代遍历（用栈）: 手动模拟递归栈，避免栈溢出
  • Morris 遍历: O(1) 空间，利用叶节点的空指针临时建立线索，遍历完恢复

思考题

为什么中序遍历 BST 是升序的？

BST 的定义：左子树所有节点 < 根 < 右子树所有节点。中序遍历先访问左子树（都比根小），再访问根，再访问右子树（都比根大）。递归地，每个子树内部也满足这个性质，所以整体是升序的。

"递归三问"中，最难的是哪一步？为什么？

第三步"怎么组合"最难。因为它需要你想清楚子问题和原问题的关系，这往往不是显然的。比如"二叉树的直径"，你需要意识到"经过当前节点的最长路径 = 左子树深度 + 右子树深度"，而"返回给父节点的"只能是单边最长路径。这种拆分需要洞察力。

Morris 遍历如何做到 O(1) 空间？代价是什么？

Morris 遍历利用叶节点本来为 null 的右指针，临时指向中序后继（建立"线索"）。这样不需要栈就能回溯。代价：(1) 会临时修改树结构（遍历完需要恢复）；(2) 遍历过程中树不是有效的BST，不能并发访问；(3) 代码复杂度更高。
