双指针技巧

双指针有两种核心形态：对撞指针和快慢指针。掌握它们能让很多 O(n^2) 问题降到 O(n)。

练习进度：已完成 0/10

1. 对撞指针（数组有序 / 两端逼近）

本质：两端向中间走，每一步根据条件决定移动哪一边。因为数组有序，移动方向能确定性地缩小搜索空间。

模板：两数之和 II

def two_sum(nums, target):
    left, right = 0, len(nums) - 1
    while left < right:
        s = nums[left] + nums[right]
        if s == target:
            return [left, right]
        elif s < target:
            left += 1
        else:
            right -= 1

为什么正确？当 s < target 时，left 已经配不上比 right 更小的数了（因为有序），所以安全丢弃 left。反之亦然。

2. 快慢指针（链表 / 找环 / 原地去重）

本质：两指针速度不同，相对位移 = 距离差。利用速度差产生有用的几何关系。

三个子模板

a) 找链表中点

slow = fast = head
while fast and fast.next:
    slow = slow.next
    fast = fast.next.next
# slow 就是中点

b) Floyd 判环

slow = fast = head
while fast and fast.next:
    slow = slow.next
    fast = fast.next.next
    if slow == fast:  # 有环
        break

c) 原地去重

def remove_duplicates(nums):
    if not nums: return 0
    slow = 0
    for fast in range(1, len(nums)):
        if nums[fast] != nums[slow]:
            slow += 1
            nums[slow] = nums[fast]
    return slow + 1
原地去重口诀：slow 指向下一个要写入的位置，fast 扫描所有元素。

3. Floyd 环检测原理

设链表结构如下：

  • 头节点到环入口距离 = a
  • 入口到相遇点距离 = b
  • 环长 = L

相遇时 slow 走了 a + b 步，fast 走了 a + b + kL 步（多绕了 k 圈）。

因为 fast = 2 * slow：

2(a + b) = a + b + kL
→ a + b = kL
→ a = kL - b

结论：从头节点和相遇点各走一步，走 a 步后一定在入口相遇。因为相遇点再走 a = kL - b 步，正好绕回入口。

4. 思考题

为什么对撞指针要求数组有序？无序时怎么办？

有序保证了单调性：移动 left 一定增大 sum，移动 right 一定减小 sum，这样每次移动都在确定性地缩小搜索空间。无序时这个性质不成立，只能用哈希表（O(n) 时间 + O(n) 空间）或先排序再双指针。

Floyd 算法中，为什么从头和相遇点同速走一定在入口相遇？

因为 a = kL - b。从相遇点出发走 a 步 = 走 kL - b 步 = 在环上绕 k 圈再退 b 步 = 正好到环入口。从头出发走 a 步也到环入口。所以两者同时到达入口。

原地去重中，如果允许最多保留 2 个相同元素怎么改？

把比较对象从 nums[fast] != nums[slow] 改为 nums[fast] != nums[slow - 1]。即 slow 往前看两个位置，只有新元素和 slow-1 不同才写入：

if fast < 2 or nums[fast] != nums[slow - 1]: slow += 1; nums[slow] = nums[fast]

更通用地，保留 k 个就比较 nums[slow - (k-1)]。
