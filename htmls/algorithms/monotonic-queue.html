单调队列 Monotonic Queue

单调队列 = 滑动窗口 + 单调栈。队尾入队清理被打败的，队头清理过期的。

核心模板

滑动窗口最大值 (Sliding Window Maximum):

from collections import deque

def max_sliding_window(nums, k):
    q = deque()  # 存下标，队头到队尾递减
    ans = []
    for i, x in enumerate(nums):
        # 1. 队尾：清理被打败的（维护单调性）
        while q and nums[q[-1]] <= x:
            q.pop()
        # 2. 入队
        q.append(i)
        # 3. 队头：清理过期的（维护窗口）
        if q[0] <= i - k:
            q.popleft()
        # 4. 取答案
        if i >= k - 1:
            ans.append(nums[q[0]])
    return ans
三步走口诀：
① 队尾维护单调性（新人打旧人）
② 队头维护窗口大小（过期的出局）
③ 队头取答案（最大/最小值在队头）

单调栈 vs 单调队列

对比项单调栈单调队列
数据结构栈（只有一端）双端队列（两端操作）
核心操作栈顶 push/pop队尾 push/pop + 队头 popleft
解决问题下一个更大/更小元素滑动窗口最值
窗口约束无（全局）有（窗口大小 k）
队头操作不需要过期淘汰
时间复杂度O(n)O(n)
一句话区别：单调队列多了一个"队头清理过期元素"的操作，因为它有窗口大小的限制。单调栈可以看作窗口大小无限的特例。

思考题

1. 为什么单调队列要存下标而不是值？
因为需要判断队头元素是否已经"过期"（滑出窗口）。只有通过下标才能判断 q[0] <= i - k。如果只存值，就无法知道这个值对应的位置是否还在窗口内。
2. 三步操作的顺序能换吗？换了会怎样？

核心约束：入队必须在清理队尾之后（否则新元素可能被自己清掉）。取答案必须在入队之后（当前元素可能就是答案）。

清理队头可以在入队前也可以在入队后（只要在取答案之前）。但习惯上放在入队后更直观：先让新人进来，再检查窗口是否超限。

如果把清理队尾放在入队之后，新元素就无法淘汰比它弱的旧人，单调性被破坏。

3. 如何用单调队列同时维护窗口内的最大值和最小值？
维护两个单调队列：一个递减队列（队头是最大值），一个递增队列（队头是最小值）。两者独立运行，各自清理各自的队尾和队头。典型题目：LC 1438（绝对差不超过限制的最长连续子数组）。
