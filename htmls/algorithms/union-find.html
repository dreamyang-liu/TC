并查集 Union-Find

每个集合用一棵树，根代表整个集合。find 找根，union 合并。

核心思想

想象一群人，每个人指向自己的"老大"。如果你要找某人属于哪个帮派，就沿着"老大"指针一直往上找，直到找到指向自己的那个人——他就是根/帮主。

两个操作：

  • find(x)：找 x 的根（帮主是谁）
  • union(x, y)：把 x 和 y 所在的集合合并（让一个帮主认另一个当老大）

最简模板（路径压缩）

class UnionFind:
    def __init__(self, n):
        self.p = list(range(n))
        self.cnt = n

    def find(self, x):
        if self.p[x] != x:
            self.p[x] = self.find(self.p[x])  # 路径压缩
        return self.p[x]

    def union(self, x, y):
        rx, ry = self.find(x), self.find(y)
        if rx == ry: return False
        self.p[rx] = ry
        self.cnt -= 1
        return True

进阶技巧

按秩合并

矮树挂到高树下面，避免退化成链。搭配路径压缩，单次操作接近 O(1)（严格说是反阿克曼函数）。

带权并查集

边上带权值，find 时把权值累加。经典题：399. 除法求值。

三大场景

  • 判连通：两点 find 相同 → 连通
  • 数连通分量：最后 cnt 就是答案
  • 离线处理（时光倒流）：删边不好做？倒着加边就行！

思考题

路径压缩具体做了什么？为什么能让 find 接近 O(1)？

路径压缩在 find 的递归回溯时，把路径上每个节点直接挂到根上。这样下次再 find 这些节点，一步就到根。多次操作后树几乎是扁平的，均摊接近 O(1)。

为什么 union 时要先 find 两个根再合并？

因为我们合并的是"集合"不是"节点"。如果不先找根，可能把中间节点连到另一棵树，导致原本同集合的节点断裂。只有根代表整个集合。

"时光倒流"技巧是怎么把删边变成加边的？

先读入所有操作，标记哪些边最终会被删。然后从最终状态开始，倒序处理：删边变成加边（union）。因为并查集只支持合并不支持拆分，倒着做刚好把拆分变成合并。经典题：LC 803。
