回溯算法 Backtracking

回溯 = 决策树 DFS：选 → 递归 → 撤销

1. 万能三件套

所有回溯问题都是同一个模板，区别只在"选择列表"怎么构造：

def backtrack(path, choices):
    if 满足结束条件:
        ans.append(path[:])   # 注意拷贝！
        return
    for c in choices:
        if not_valid(c): continue   # 剪枝
        path.append(c)              # 做选择
        backtrack(path, new_choices) # 递归
        path.pop()                  # 撤销选择
核心思想：在决策树上做 DFS，每到一个节点就做选择，递归进入下一层，回来时撤销选择（恢复现场），这就是"回溯"。

2. 三大问题族

子集 (Subsets)

每个元素"选"或"不选"，或者用枚举起点版（for i in range(start, n)）。结果数量：2^n

排列 (Permutations)

顺序敏感！[1,2] 和 [2,1] 是不同答案。需要 used[] 数组标记哪些元素已经用过。结果数量：n!

组合 (Combinations)

固定长度 k，从 start 开始枚举避免重复。结果数量：C(n,k)

速记：子集=所有节点收集，排列=叶子节点+used，组合=叶子节点+start

3. 去重套路

步骤：先排序，同一层跳过相同值。

nums.sort()
def backtrack(start, path):
    ...
    for i in range(start, len(nums)):
        if i > start and nums[i] == nums[i-1]:
            continue  # 同一层去重
        ...
口诀：同一层不能选相同（兄弟去重），同一路径可以（父子可同）。
"兄弟"就是 for 循环的同一轮，"父子"就是递归的上下层。

4. 不写错要点

  • path[:] 拷贝 — 不拷贝的话 path 最终会变成空列表
  • 状态恢复对称 — append 对 pop，used[i]=True 对 used[i]=False
  • 排列用 used[]，组合用 start — 别搞混
  • 去重前必须排序

7. 思考题

Q1: 为什么 ans.append(path[:]) 而不是 ans.append(path)？

path 是一个列表引用，在回溯过程中会不断 append/pop 修改。如果直接存引用，最终 ans 里所有元素都指向同一个列表（而且回溯结束后 path 是空的）。path[:] 创建一个快照副本，冻结当前状态。

Q2: 子集问题用"选/不选"和"枚举起点"两种写法，哪种更通用？为什么？

枚举起点版更通用。原因：
1. 它能直接扩展到"组合""组合总和"等变种（加一个长度限制就是组合）
2. 去重逻辑（i > start and nums[i] == nums[i-1]）更自然
3. "选/不选"版本在需要剪枝或去重时反而更难改
"选/不选"更适合理解本质（二叉决策树），实际刷题用枚举起点版。

Q3: 去重时 if i > start and nums[i] == nums[i-1]: continue 为什么是 i > start 而不是 i > 0？

i > start 确保只在"同一层"去重。当 i == start 时，这是当前层的第一个选择，即使和前面值相同也必须保留（因为前面那个值是"父节点"选的，不是"兄弟"选的）。
如果用 i > 0，会误删"父子路径上值相同"的合法情况。比如 [1,1,2] 求子集，[1,1] 是合法子集，用 i > 0 会跳过第二个1导致丢失这个答案。
