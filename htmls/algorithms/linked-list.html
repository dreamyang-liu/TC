6个核心操作

反转链表 — prev/cur 双指针，口诀：存转移（存nxt → cur.next=prev → prev=cur, cur=nxt）
找中点 — 快慢指针，slow一步fast两步，fast到尾时slow在中间
判环 + 找入口 — Floyd：快慢指针相遇后，一个回起点同速走再次相遇即入口
合并两个有序链表 — dummy + tail，比较后接上小的
删除倒数第N个 — dummy + 快慢指针拉开n步
K个一组反转 — 先数k个，反转区间，递归/迭代拼接

不写错要点

dummy 是金科玉律 — 头节点可能被删/换时，用 dummy 统一处理

画图画图画图 — 不画图直接写指针操作 = 必错

空指针检查顺序 — 先检查 node 非空，再访问 node.next

反转模板

def reverseList(head):
    prev, cur = None, head
    while cur:
        nxt = cur.next      # 存
        cur.next = prev     # 转
        prev = cur          # 移
        cur = nxt           # 移
    return prev

思考题

为什么链表题几乎都需要 dummy 节点？什么时候可以不用？

当头节点可能被删除或更换时（如合并、删除、反转），没有dummy就要对头节点特殊判断。如果确定头节点不会变（如只是遍历或修改值），可以不用dummy。

K个一组反转中，为什么要先检查后续是否还有k个？

题目要求不足k个的尾部保持原序。如果不先检查就反转，最后一组不足k个也被反转了，不符合要求。先数k个确认够了再反转。

如何不用额外空间判断链表是否为回文？

组合三个操作：(1) 快慢指针找中点 → (2) 反转后半段 → (3) 双指针从头和中间同时比较。用到了找中点 + 反转两个核心操作。
