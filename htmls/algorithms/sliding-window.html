滑动窗口

右指针扩张，左指针在窗口不合法时收缩

练习进度: 已完成 0/8

最简模板（最长合法窗口）

def sliding_window(s):
    from collections import Counter
    window = Counter()
    left = 0
    ans = 0
    for right, c in enumerate(s):
        window[c] += 1              # 右指针进窗口
        while not_valid(window):     # 窗口不合法
            window[s[left]] -= 1    # 左指针出窗口
            if window[s[left]] == 0:
                del window[s[left]]
            left += 1
        ans = max(ans, right - left + 1)  # while 外更新答案
    return ans

两种变体

最长合法窗口

while 不合法 - 收缩到合法为止，答案在 while 外更新

场景: 最长无重复子串、最多含 K 个 0 的最长连续 1

最短合法窗口

while 合法 - 只要合法就记录答案并继续收缩，答案在 while 内更新

场景: 和 >= target 的最短子数组、覆盖所有字符的最小窗口

不写错要点

  • 进出对称: right 进窗口做了什么操作，left 出窗口就要做对应的逆操作
  • right 每轮 +1: for 循环保证，不需要手动管理
  • left 只在 while 里动: 绝不在 while 外面移动 left

"恰好 K" = "至多 K" - "至多 K-1"

def exactly_k(nums, k):
    return at_most(nums, k) - at_most(nums, k - 1)

def at_most(nums, k):
    # 标准滑窗，统计所有"至多 K 个不同字符"的子数组数
    left = 0
    count = {}
    ans = 0
    for right, x in enumerate(nums):
        count[x] = count.get(x, 0) + 1
        while len(count) > k:
            count[nums[left]] -= 1
            if count[nums[left]] == 0:
                del count[nums[left]]
            left += 1
        ans += right - left + 1  # 以 right 结尾的合法子数组数
    return ans

思考题

为什么左指针只向右移动，不会回退？这保证了什么复杂度？

因为当窗口不合法时，我们需要缩小窗口（左指针右移）。而右指针扩张后窗口变大，只可能"更不合法"，所以左指针没有理由回退。左右指针各自最多移动 n 次，所以总时间复杂度是 O(2n) = O(n)，而不是 O(n^2)。这就是滑动窗口比暴力枚举快的本质原因。

"恰好 K 个不同字符" 为什么不能直接滑窗？怎么转化？

直接滑窗的问题：窗口中恰好有 K 个不同字符时，既不能确定要扩张还是收缩。扩张可能变成 K+1 个（不合法），收缩可能变成 K-1 个（也不合法）。没有单调性，无法用 while 循环维护。

转化方法：恰好 K = 至多 K - 至多 (K-1)。"至多 K" 有单调性（窗口越大越可能超过 K），可以标准滑窗。

如果窗口合法性判断是 O(1) 的，整体复杂度是多少？为什么？

整体 O(n)。虽然有 for + while 两层循环看起来像 O(n^2)，但分析总操作次数：right 走了 n 步，left 总共也最多走 n 步（只增不减），每步的合法性判断 O(1)，所以总计 O(n) + O(n) = O(n)。这是「均摊分析」的经典例子。
