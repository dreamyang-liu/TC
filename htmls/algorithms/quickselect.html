快速选择 Quickselect

快排只递归 K 所在的那一半。期望 O(n)，最坏 O(n^2)。

核心模板

import random

def quickselect(nums, k):
    def partition(lo, hi):
        p = random.randint(lo, hi)
        nums[p], nums[hi] = nums[hi], nums[p]
        pivot = nums[hi]
        i = lo
        for j in range(lo, hi):
            if nums[j] < pivot:
                nums[i], nums[j] = nums[j], nums[i]
                i += 1
        nums[i], nums[hi] = nums[hi], nums[i]
        return i

    lo, hi = 0, len(nums) - 1
    while True:
        p = partition(lo, hi)
        if p == k: return nums[p]
        elif p < k: lo = p + 1
        else: hi = p - 1

三路划分（应对重复元素）

当数组有大量重复元素时，普通 partition 退化。三路划分将数组分为 <pivot, =pivot, >pivot 三段。

def partition3(nums, lo, hi):
    pivot = nums[random.randint(lo, hi)]
    lt, gt, i = lo, hi, lo  # lt: <pivot 的右边界, gt: >pivot 的左边界
    while i <= gt:
        if nums[i] < pivot:
            nums[lt], nums[i] = nums[i], nums[lt]
            lt += 1; i += 1
        elif nums[i] > pivot:
            nums[gt], nums[i] = nums[i], nums[gt]
            gt -= 1
        else:
            i += 1
    return lt, gt  # [lt, gt] 区间全是 pivot

思考题

为什么要随机化 pivot？不随机化最坏情况是什么？

如果每次都选第一个或最后一个元素作为 pivot，遇到已排序数组时，每次 partition 只能排除一个元素，退化为 O(n^2)。随机化使得最坏情况的概率极低，期望复杂度稳定在 O(n)。

快速选择的期望时间为什么是 O(n) 而不是 O(n log n)？

快排需要递归两边，总工作量是 n + n + ... = O(n log n)。但快选只递归一边，每次问题规模期望缩小一半：n + n/2 + n/4 + ... = 2n = O(n)。这是一个等比级数收敛的结果。

什么情况下用堆比快选更合适？

1) 数据是流式到达的（在线），无法一次拿到全部数据；2) 需要动态维护 Top K（频繁插入/删除）；3) 需要稳定的最坏时间复杂度保证；4) k 远小于 n 时，O(n log k) 和 O(n) 差距不大但堆更简单。
