区间 DP

dp[i][j] = 区间 [i, j] 上的最优答案，长度从小到大枚举

核心思想

区间 DP 的本质：大区间的答案依赖小区间，所以必须先算短区间再算长区间。

dp[i][j] 表示对子数组 [i..j] 做某种操作的最优值。转移时枚举分割点 k，把 [i,j] 拆成 [i,k] 和 [k+1,j]。

模板代码

# 经典区间 DP 模板（按长度枚举）
for length in range(2, n + 1):        # 区间长度从 2 到 n
    for i in range(n - length + 1):   # 左端点
        j = i + length - 1            # 右端点
        for k in range(i, j):         # 分割点
            dp[i][j] = max(dp[i][j],
                dp[i][k] + dp[k+1][j] + cost(i, k, j))

三种遍历顺序

1. 按长度枚举（最直观）：先算所有长度=2的区间，再算长度=3…最后算长度=n

2. i 倒序 + j 正序：因为 dp[i][j] 依赖 dp[i][k]（i不变,k<j）和 dp[k+1][j]（k+1>i），所以 i 从大到小、j 从小到大也行

3. 记忆化递归：最灵活，不用想顺序，直接递归 + cache

三大类问题

1. 合并/切分型（石子合并）

把相邻的东西合并（或把整体切分），每次操作有代价，求最小/最大总代价。

2. 回文/对称型（最长回文子序列）

dp[i][j] 表示 s[i..j] 内最长回文子序列长度。s[i]==s[j] 时从 dp[i+1][j-1] 转移。

3. 反向枚举型（戳气球）

戳气球精髓：

正着想"先戳哪个" → 戳完一个后，左右气球变成相邻了，状态混乱不可控

倒着想"最后戳哪个" → 最后戳 k 时，左右两边是独立的子问题！

核心：正难则反，让最后一步操作把区间切成不相关的两半

动画：石子合并

数组 [3, 1, 5, 2]，求合并所有石子的最小代价。每次合并相邻两堆，代价 = 两堆石子总数。

思考题

1. 为什么区间 DP 必须按长度从小到大？ ▶
因为 dp[i][j] 的计算依赖 dp[i][k] 和 dp[k+1][j]，这两个区间长度都小于 [i,j]。如果长区间先算，所依赖的短区间还没有值，结果就错了。本质就是拓扑序：先算被依赖的，再算依赖别人的。
2. 戳气球问题中，"最后戳" vs "先戳" 的状态为什么不同？ ▶
"先戳 k" → 戳完 k 后，k 左右的气球变成相邻，后续操作的收益跟这次选择耦合在一起，无法拆分成独立子问题。"最后戳 k" → 在 k 被戳的那一刻，区间 (i,k) 和 (k,j) 的气球早已被戳光，k 左边是边界 i，右边是边界 j，收益确定为 nums[i]*nums[k]*nums[j]，两侧完全独立。
3. 记忆化递归在区间 DP 中有什么优势？ ▶
优势：(1) 不需要想清楚填表顺序，递归自动保证先算子问题；(2) 有些区间可能永远不会被用到，记忆化可以跳过（虽然区间 DP 通常都会用到）；(3) 代码更直观，思路对应"自顶向下分治"。缺点：递归栈深度 O(n)，对 n 很大时可能栈溢出。
