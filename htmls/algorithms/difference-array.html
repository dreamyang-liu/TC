差分数组 / 前缀和

用变化量代替全量。如果整体=个体之和，且变化量稀疏，就在变化量层面合并。

核心思想（First Principles）

想象你管理一排路灯的亮度。老板说"把第2到第5盏灯都调亮3"。你可以一盏一盏调（O(n)），也可以只记录"从第2开始+3，从第6开始-3"（O(1)）。后者就是差分的思想。

两个关键性质

  • 性质A 线性性: 和的变化量 = 变化量的和。多次操作可以在差分层合并，最后一次性还原。
  • 性质B 局部性: 每个操作 gᵢ 的变化量只在 O(1) 个点非零（区间加只影响两个端点）。

算法步骤

  1. 稀疏编码: 每个操作 gᵢ → O(1) 个 delta 事件（diff[l]+=c, diff[r+1]-=c）
  2. 实例化: 前缀和得到最终数组 f(x)
  3. 求目标: 扫一遍找 min/max/满足条件的值

套路族

  • 一阶差分 区间加常数 — diff[l]+=c, diff[r+1]-=c
  • 二阶差分 区间加等差数列 — 需要两层差分
  • 扫描线 二维差分的变种
  • 懒标记线段树 差分思想的树上推广

判题流程

能写成 f(x) = Σgᵢ(x) 吗？

→ 看 f 整体结构（单调/单峰）

→ 看 gᵢ 局部结构（分段常数 → 一阶差分，分段线性 → 二阶差分）

一句话总结：能用变化量描述的，就不要用绝对值描述。

思考题

差分和前缀和是什么关系？

互逆操作。差分是"求导"（相邻元素之差），前缀和是"积分"（累加）。对一个数组先差分再前缀和，得到原数组。反之亦然。

为什么 diff[l]+=c, diff[r+1]-=c 能实现区间加？

前缀和时，diff[l]的+c会"传播"到l及之后的所有位置。而diff[r+1]的-c会从r+1开始抵消这个+c。所以只有[l, r]范围内的元素被加了c。这就是差分的精髓：用两个端点的变化来描述一整段的变化。

什么时候该用差分，什么时候该用线段树？

差分：所有修改先做完，最后一次性查询（离线）。线段树：修改和查询交替进行（在线）。差分是 O(n) 空间 O(1) 修改 O(n) 查询；线段树是 O(n) 空间 O(log n) 修改 O(log n) 查询。如果能离线处理，差分更简单高效。
