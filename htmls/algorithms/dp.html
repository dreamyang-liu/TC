动态规划 (Dynamic Programming)

动态规划 = 带记忆化的递归。把重叠子问题算一次存起来。

已完成 0/10

1. DP 四步走

做任何 DP 题，按这个顺序想：

  1. 定义状态 - dp[i] 代表什么？（最关键的一步）
  2. 写转移方程 - dp[i] 跟谁有关？怎么算？
  3. 初始化 - 边界条件是什么？
  4. 遍历顺序 - 从小到大？从大到小？
记住：状态定义决定一切。状态定义对了，转移方程自然就出来了。

2. 一维 DP

爬楼梯型（线性递推）

当前状态只依赖前面有限个状态：

# 爬楼梯: 每次1步或2步，到第n阶有多少种走法
dp[i] = dp[i-1] + dp[i-2]
# 初始化: dp[0]=1, dp[1]=1

最长递增子序列 (LIS)

dp[i] = 以 nums[i] 结尾的最长递增子序列长度：

# O(n^2) 做法
for i in range(n):
    dp[i] = 1
    for j in range(i):
        if nums[j] < nums[i]:
            dp[i] = max(dp[i], dp[j] + 1)
ans = max(dp)
LIS 还有 O(n log n) 的贪心+二分做法，面试时两种都要会。

3. 二维 DP

最长公共子序列 (LCS)

dp[i][j] = text1 前 i 个字符和 text2 前 j 个字符的 LCS 长度：

if text1[i-1] == text2[j-1]:
    dp[i][j] = dp[i-1][j-1] + 1    # 相等看左上角+1
else:
    dp[i][j] = max(dp[i-1][j], dp[i][j-1])  # 不等看左和上的max

LCS 填表动画

字符串: "ABCDE" vs "ACE"

编辑距离

dp[i][j] = word1 前 i 个变成 word2 前 j 个的最少操作数：

if word1[i-1] == word2[j-1]:
    dp[i][j] = dp[i-1][j-1]        # 不用操作
else:
    dp[i][j] = 1 + min(
        dp[i-1][j],    # 删除
        dp[i][j-1],    # 插入
        dp[i-1][j-1]   # 替换
    )

4. 背包问题

0/1 背包

每个物品只能用一次。一维优化时容量倒序遍历：

for i in range(n):           # 物品外层
    for j in range(W, w[i]-1, -1):  # 容量倒序!
        dp[j] = max(dp[j], dp[j - w[i]] + v[i])

完全背包

每个物品可以用无限次。一维优化时容量正序遍历：

for i in range(n):           # 物品外层
    for j in range(w[i], W+1):      # 容量正序!
        dp[j] = max(dp[j], dp[j - w[i]] + v[i])

能否装满dp[0]=Truedp[j] = dp[j] or dp[j-w]

5. 记忆化递归

本质上和迭代 DP 等价，但写起来更像递归思维：

from functools import cache

@cache
def dfs(i, j):
    if i == 0 or j == 0:
        return 0
    if text1[i-1] == text2[j-1]:
        return dfs(i-1, j-1) + 1
    return max(dfs(i-1, j), dfs(i, j-1))

# 调用后记得清缓存: dfs.cache_clear()
记忆化递归的好处：不用纠结遍历顺序，递归自然保证依赖顺序。缺点：递归栈深度有限制（Python默认1000），常数大一些。

思考题

Q1: 为什么 0/1 背包容量要倒序？如果正序会怎样？

点击查看提示

正序的话，dp[j-w[i]] 可能已经被本轮更新过了，相当于物品 i 被用了多次。倒序保证引用的是"上一轮"的值，即物品 i 还没被选的状态。

Q2: 记忆化递归和迭代 DP 的优劣分别是什么？

点击查看提示

记忆化递归：不用想遍历顺序、只算需要的子问题（稀疏时快）、代码直观。迭代 DP：没有栈溢出风险、常数小、方便空间优化（滚动数组）。

Q3: "排列"和"组合"在背包中遍历顺序为什么不同？

点击查看提示

物品在外层：每个物品按顺序考虑，(1,2) 和 (2,1) 只会出现一种 = 组合。容量在外层：对每个容量都考虑所有物品，同样的物品可以以不同顺序被选 = 排列。
