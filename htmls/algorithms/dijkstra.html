Dijkstra 最短路径算法

带优先队列的BFS：每次取当前已知距离最小的未确定节点去松弛邻居。只适用于非负边权。

模板代码

import heapq
def dijkstra(n, edges, src):
    g = [[] for _ in range(n)]
    for u, v, w in edges: g[u].append((v, w))
    dist = [float('inf')] * n
    dist[src] = 0
    pq = [(0, src)]
    while pq:
        d, u = heapq.heappop(pq)
        if d > dist[u]: continue  # 跳过过期条目
        for v, w in g[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(pq, (nd, v))
    return dist

三个关键点

1 堆里存 (距离, 节点) 这个元组，保证 heappop 按距离排序

2 弹出时检查 if d > dist[u]: continue ——跳过过期的旧条目

3 松弛成功后再 push 新的 (nd, v)，不要提前 push

思考题

为什么 Dijkstra 不能处理负权边？给一个反例。

Dijkstra 贪心地"确定"当前最小距离节点，一旦确定就不再更新。如果后面有负权边能让已确定节点距离更短，算法无法回头修正。反例：A→B(1), A→C(5), C→B(-10)，从A出发先确定B=1，但实际最短是A→C→B=-5。

if d > dist[u]: continue 这一行的作用是什么？不加会怎样？

堆中可能有同一节点的多个旧条目。这行跳过过期条目，避免用旧距离去松弛邻居。不加不影响正确性，但会做大量无用松弛，时间复杂度退化。

0-1 BFS 为什么比 Dijkstra 更快（边权只有0/1时）？

0-1 BFS 用双端队列：权0的边加到队头，权1的边加到队尾。不需要堆操作，时间 O(V+E)，而 Dijkstra 是 O((V+E)logV)。
