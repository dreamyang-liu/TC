BFS 广度优先搜索

一层一层往外扩。第一次访问到某点的步数 = 最短步数（边权为1）。

1. 最简模板（按层遍历）

from collections import deque

def bfs(start):
    q = deque([start])
    visited = {start}
    step = 0
    while q:
        for _ in range(len(q)):
            node = q.popleft()
            if is_target(node): return step
            for nxt in neighbors(node):
                if nxt not in visited:
                    visited.add(nxt)
                    q.append(nxt)
        step += 1
    return -1

2. 关键点

入队时就标记 visited（不是出队时！）否则同一个节点可能被多次入队，导致超时或错误结果。

3. 网格 BFS

方向数组：dirs = [(-1,0),(1,0),(0,-1),(0,1)]
判断越界 + 判断障碍 + 判断是否已访问

4. 多源 BFS

所有起点一开始就塞进队列，相当于加了一个"超级源点"连接所有起点。典型题：腐烂的橘子、01矩阵。

5. 双向 BFS

从起点和终点同时扩散，每次扩展较小的那一边。时间从 O(b^d) 降到 O(b^(d/2))，其中 b 是分支因子，d 是深度。

6. BFS vs DFS 选择

  • 最短步数 / 最少操作次数 → BFS
  • 所有路径 / 排列组合 / 连通分量 → DFS
  • 拓扑排序 → BFS (Kahn) 或 DFS 都行

思考题

为什么 BFS 能保证找到最短路径（边权为1时）？

因为 BFS 按层扩散，第 k 层的节点离起点的距离恰好是 k。第一次到达目标节点时，经过的层数就是最小步数。后续再到达目标的路径，层数只会更大或相等，不会更小。

"入队时标记" vs "出队时标记" 有什么区别？

入队时标记：一个节点只会入队一次，队列大小 = O(节点数)。
出队时标记：同一个节点可能被多个邻居同时入队，队列可能膨胀到 O(边数)，在网格题中容易超时（比如 4 个邻居都把同一个点入队了）。对于无权图求最短路，两者结果一样，但后者浪费时间和空间。

双向 BFS 的时间复杂度优势是什么？什么时候适合用？

单向 BFS 搜索空间为 O(b^d)，双向 BFS 为 O(2 * b^(d/2))，指数级缩小。
适用条件：已知起点和终点，且能反向扩展邻居（如 Word Ladder）。不适合：终点不明确或无法反向搜索的场景。
